## תרשים D3 — EndOfDay1
### הסבר לוגיקת טיפול באירוע סיום יום ראשון

### מה מייצג האירוע?

אירוע EndOfDay1 מופעל פעם אחת בלבד בסוף יום ראשון.
הוא אחראי על שלושה דברים: זריעת יום 2 לפני שמשהו אחר קורה, צילום מדדי הביצועים של יום 1, וסריקת הישויות שנמצאות בתור או במעבר ומחליטה עבורן אם הן נשארות ללינה או עוזבות.

### הסבר צומת אחר צומת

**צומת 1 — תזמון הגעות ביום 2 והופעות הפתיחה**

הדבר הראשון שהאירוע עושה הוא לתזמן את יום 2: הגעות זוגות, הגעות יחידים, הופעת הפתיחה בבמה הראשית והופעת הפתיחה בבמה המשנית. קבוצות חברים אינן מוזמנות ביום 2 — הן מגיעות רק ביום ראשון לפי המפרט.

```python
sim.seed_arrivals(2)
sim.schedule(ShowStart_MainStage(sim.day_open(2)))
sim.schedule(ShowStart_SideStage(sim.day_open(2)))
```

**צומת 2 — צילום נתוני ביצועים של יום 1**

לפני שמתחיל עיבוד הישויות, נשמרים כל מדדי הביצועים של יום 1. חשוב לעשות זאת כאן — לפני שישויות מסומנות כ"עזבו" ולפני שנתוני יום 2 מתחילים להצטבר.

```python
sim.kpi.snapshot_day1()
```

**צומת 3 — לולאה: ישויות בתור או במעבר בלבד**

הסימולציה מזהה את כל הישויות שנמצאות כרגע בתור לפעילות כלשהי, או שסיימו פעילות ועוד לא הצטרפו לתור הבא. ישויות שנמצאות בהופעה, אוכלות או מקבלות שירות — ממשיכות לבד ואין צורך לטפל בהן כאן.

```python
for ent in list(sim.entities):
    if ent.current_position in ("queueing", "transit"):
        sim.handle_overnight(ent)
```

**צומת 4 — נותרה ישות שלא טופלה בתור או במעבר?**

לפני כל עיבוד נבדק האם עדיין יש ישות שלא קיבלה טיפול. אם אין — הלולאה נגמרת והאירוע מסתיים.

**צומת 5 — הסר את הישות מכל תור ופעילות**

לפני שמחליטים מה הישות עושה, היא מוצאת מכל תור ופעילות שהיא מחוברת אליה. זה קורה **לפני** החלטת הלינה, כדי שהמצב יהיה נקי לפני שמסמנים אותה.

```python
self.detach(entity)   # מסיר מכל תור, במה, וסט של ישויות בשירות
```

**צומת 6 — מהו סוג הישות**

כל סוג ישות מחליט על לינה לפי חוק שונה. הקריאה `decide_overnight()` היא פולימורפית — כל ישות יודעת את החוק שלה.

**ענף קבוצת חברים — האם רכשו לינה מראש?**

קבוצת חברים מחליטה על לינה כבר בעת הכניסה לפסטיבל, לא בסוף היום. ב-t=0 נדגם עבורה Bernoulli(0.7) ונשמר בדגל `bought_lodging`. ההחלטה אינה תלויה בחוויה.

```python
def decide_overnight(self):   # FriendsGroup
    return "stay" if self.bought_lodging else "leave"
```

**ענף זוג — האם ממוצע שביעות הרצון גדול מ-7?**

זוג מחליט בסוף יום 1 לפי חוויית היום. מחשבים את הממוצע של שביעות הרצון של **שני** בני הזוג.

```python
def decide_overnight(self):   # Couple
    return "stay" if self.mean_satisfaction() > 7 else "leave"
```

**ענף יחיד — תמיד עוזב**

יחידים הם מבקרי יום בלבד לפי המפרט. אין להם אפשרות לינה.

```python
def decide_overnight(self):   # Single
    return "leave"
```

**צומת 7 — סמן "לינה", גבה תשלום לזוגות, תזמן המשך ביום 2**

ישות שמחליטה להישאר מסומנת כ"לינה". אם מדובר בזוג, נגבים ממנו דמי לינה של 250 ₪ לאדם. לאחר מכן מתוזמן אירוע Day2Resume שיחזיר אותה לפעילות בתחילת יום 2 עם מסלול חדש.

קבוצת חברים אינה משלמת כאן — היא כבר שילמה את חבילת הלינה בעת הכניסה.

```python
entity.current_position = "night"
if entity.entity_type == "Couple":
    self.kpi.record_revenue("lodging", 250 * entity.size)
self.schedule(Day2Resume(self.day_open(2), entity))
```

**צומת 8 — סמן "עזב", תזמן יציאה מהפסטיבל**

ישות שמחליטה לעזוב מתוזמן עבורה אירוע EndOfStay שירשום את יציאתה.

```python
self.schedule(EndOfStay(self.clock, entity))
```

לאחר כל אחד מהצמתים 7 ו-8 — חוזרים לצומת 4 ומטפלים בישות הבאה.
